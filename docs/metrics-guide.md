## 2. Screener

### Что это
Таблица всех 30 символов с z-scores по 4 метрикам и средним перцентилем. Быстрый скан рынка для поиска аномалий.

### Колонки

| Колонка | Описание |
|---------|----------|
| Symbol | Тикер (BTCUSDT, ETHUSDT, ...) |
| Price | Текущая цена (Binance) |
| OI z | Z-score открытого интереса |
| Fund z | Z-score фандинга |
| Liq z | Z-score ликвидаций (абс. значение дельты) |
| Vol z | Z-score объёма |
| Pct Avg | Среднее 4 перцентилей |
| OI Δ24h | Изменение OI за 24ч (%) |
| Price Δ24h | Изменение цены за 24ч (%) |

### Цветовая кодировка z-scores

```
z ≥ +2   → красный    (экстремум вверх)
z ≥ +1   → жёлтый     (повышенный)
z ≤ -1   → синий      (пониженный)
z ≤ -2   → зелёный    (экстремум вниз)
иначе    → серый      (норма)
```

### Как читать
- **Pct Avg ≥ 80** — символ в экстремальном состоянии по нескольким метрикам, требует внимания
- **Pct Avg ≥ 60** — повышенная активность, потенциальный кандидат
- Сортировка по `abs(z)` — выносит наверх самые аномальные значения

### Стратегия: Выбор кандидатов

```
Шаг 1: Отсортировать по Pct Avg (desc)
Шаг 2: Выбрать символы с Pct Avg ≥ 60
Шаг 3: Проверить дивергенции:

  ┌─────────────────────────────────────────────────┐
  │  OI_z высокий + Fund_z низкий = накопление      │
  │  OI_z высокий + Fund_z высокий = перегрев       │
  │  OI_z падает  + Price растёт = дивергенция (!)  │
  │  Vol_z > 2    + любой другой z > 1 = breakout   │
  └─────────────────────────────────────────────────┘

Шаг 4: Перейти в Detail для подтверждения
```

## 3. Open Interest

### Что это
Суммарный открытый интерес в USD по 4 биржам. Показывает сколько денег залочено в позициях. Растущий OI = новые позиции открываются, падающий = позиции закрываются.

### Агрегация

```
total_oi = binance_oi_usd
         + bybit_oi_coins  × binance_price
         + okx_oi_coins    × binance_price
         + bitget_oi_usd
```

### Как читать

```
         OI ($M)
    200 ┤
        │         ╭──╮
    180 ┤    ╭───╯   │     ← OI растёт: новые позиции
        │   ╭╯        │
    160 ┤──╯          ╰──── ← OI падает: ликвидации / закрытие
        │
    140 ┤
        └──────────────────── время

  Z-Score:
   +2 ┤ · · · · · · · · · ·  ← экстремум: перегрев рынка
   +1 ┤ - - - - - - - - - -
    0 ┤────────────────────── ← среднее (365 дней)
   -1 ┤ - - - - - - - - - -
   -2 ┤ · · · · · · · · · ·  ← экстремум: вымытые позиции
```

### Стратегия: OI/Price дивергенция

```
Сценарий 1: Цена ↑ + OI ↑ = подтверждённый тренд
──────────────────────────────────────────────────
  Price ──╱──╱──╱───  OI ──╱──╱──╱───
  → Тренд здоровый, удерживать позицию

Сценарий 2: Цена ↑ + OI ↓ = дивергенция (разворот)
──────────────────────────────────────────────────
  Price ──╱──╱──╱───  OI ──╲──╲──╲───
  → Рост на закрытии шортов, топ близко
  → Искать шорт при касании наклонной сверху

Сценарий 3: Цена ↓ + OI ↑ = накопление шортов
──────────────────────────────────────────────────
  Price ──╲──╲──╲───  OI ──╱──╱──╱───
  → Агрессивные шорты, потенциальный шорт-сквиз
  → Готовиться к лонгу на сквиз-уровнях

Сценарий 4: OI_z > +2 = перегрев
──────────────────────────────────────────────────
  → Рынок перегружен позициями
  → Каскадные ликвидации вероятны
  → Не открывать новые позиции, ждать разгрузку
```

### Привязка к волновой теории

```
  Волна 1: OI начинает расти с низов (z < -1)
  Волна 3: OI резко ускоряется (z переходит в +1..+2)
  Волна 5: OI на максимуме (z > +2), дивергенция с ценой
  Коррекция A: OI резко падает, каскад ликвидаций
```

## 4. Funding Rate

### Что это
Периодическая выплата между лонгами и шортами для удержания перп-цены у спота. Положительный фандинг = лонги платят шортам (рынок перекуплен). Отрицательный = шорты платят лонгам.

### Settlement cycles

| Биржа | Период | Нормализация |
|-------|--------|-------------|
| Binance, Bybit, OKX, MEXC | 8ч | raw |
| Paradex, EdgeX, Aster, Variational | 8ч | raw |
| Hyperliquid | 1ч | × 8 |
| Lighter | 1ч | × 8 |
| Extended (Starknet) | 1ч | × 8 |

**Annualized:** `rate × 3 × 365 × 100%` (3 сеттлмента/день × 365 дней)

### Как читать

```
  Funding Rate (8h)
  +0.05% ┤                 ╭─╮
         │              ╭──╯ │    ← лонги платят: рынок перегрет
  +0.01% ┤─ ─ ─ ─ ─ ─╱─ ─ ─ ─ ─  ← нейтраль (0.01% = стандарт)
       0 ┤───────────/────────╰──
  -0.01% ┤─ ─ ─ ─ ╱─ ─ ─ ─ ─ ─
         │       ╭╯              ← шорты платят: рынок перепродан
  -0.05% ┤──────╯
         └──────────────────────── время
```

### Пороги алертов

| Порог | Значение (8ч) | APR | Действие |
|-------|--------------|-----|----------|
| Extreme | ≥ 0.3% | ≥ 328% | Алерт `FUNDING_EXTREME` |
| Spread | ≥ 0.2% между биржами | — | Алерт `FUNDING_SPREAD` (арб) |

### Стратегия: Mean Reversion на экстремумах

```
  Funding Z-Score
  +2 ┤ · · · · · · · ╭─╮· ·  ← ШОРТ ЗОНА: funding_z > +2
     │              ╱   ╰╮     экстремально положительный фандинг
  +1 ┤ - - - - - -╱- - - ╰─   = рынок перекуплен
   0 ┤────────────────────────
  -1 ┤ - - - - - - - - - - -
     │
  -2 ┤ · · · · · · · · · · ·  ← ЛОНГ ЗОНА: funding_z < -2
     └────────────────────────

  Правила:
  1. Funding_z > +2 → ищем шорт (лонги перегружены, платят много)
  2. Funding_z < -2 → ищем лонг (шорты перегружены)
  3. Подтверждение: совпадение с наклонной/зоной интереса на графике
  4. Фильтр: не торговать funding_z в изоляции — только с контекстом
```

### Стратегия: Funding Арбитраж

```
  Биржа A:  +0.08% (платят лонги)
  Биржа B:  -0.02% (платят шорты)
  Спред:     0.10%
  ──────────────────────────────
  Действие:
  → Шорт на A (получаем +0.08%)
  → Лонг на B (получаем +0.02%)
  → Дельта-нейтральная позиция
  → Доход: 0.10% за 8ч = ~136% APR
  → Вкладка Funding → SpreadTable для мониторинга
```

## 5. Liquidations

### Что это
Реал-тайм ликвидации с Binance и Bybit через WebSocket. Delta = long_liq - short_liq. Каскады ликвидаций создают точки разворота.

### WebSocket источники

| Биржа | Endpoint | Side mapping |
|-------|---------|-------------|
| Binance | `!forceOrder@arr` | SELL = long liquidated |
| Bybit | `allLiquidation` | Sell = long liquidated |

**Batch flush:** 3 события ИЛИ 10 секунд (что раньше).

### Как читать

```
  Liq Delta ($M)
  +5 ┤          │         ← массовые ликвидации лонгов
     │          │            (принудительные продажи)
   0 ┤──────────┼──────────
     │     │    │    │
  -5 ┤     │         │    ← массовые ликвидации шортов
     │     │         │       (принудительные покупки = поддержка)
     └──────────────────── время

  Интерпретация delta:
  │ delta > 0  │ больше лонгов ликвидировано │ давление продаж │
  │ delta < 0  │ больше шортов ликвидировано │ давление покупок│
  │ |delta|↑↑  │ каскад ликвидаций           │ разворот близко │
```

### Стратегия: Каскады как точки разворота

```
  Price
  100k ┤──╮
       │  │
   95k ┤  ╰──╮
       │     │    Каскадные ликвидации лонгов
   90k ┤     ╰──╮ ←─────────────────────────
       │        │  liq_z > +2, delta резко +
   85k ┤        ╰── выжигание завершено
       │             ╭──╮
   87k ┤            ╯   ╰── разворот вверх
       └─────────────────────────────────── время

  Правила:
  1. liq_z > +2 + delta резкий всплеск → ликвидационный каскад
  2. После каскада: вся слабость выжжена
  3. Ищем лонг на зоне интереса/наклонной ПОСЛЕ каскада
  4. Не во время каскада! Дождаться завершения (delta → 0)
```

### Привязка к волновой теории

```
  Волна 3 вниз: каскад ликвидаций лонгов (delta >> 0)
  Волна C:      финальный каскад, максимальная дельта
  Точка входа:  после волны C, когда дельта нормализуется
  Подтверждение: OI упал (z < -1), funding стал отрицательным
```

## 6. Volume

### Что это
Торговый объём в USD (Binance). Z-score показывает аномалии — всплески объёма подтверждают breakout или кульминацию.

### Как читать

```
  Volume ($B)
  10 ┤       │
     │       │         ← аномальный объём (vol_z > +2)
   5 ┤  │    │    │       breakout или кульминация
     │  │    │    │
   2 ┤──┼────┼────┼──── ← средний объём
     │  │    │    │
   0 ┤──┴────┴────┴──── время
```

### Стратегия: Volume Confirmation

```
  ┌────────────────────────────────────────────────┐
  │  vol_z > +2 + пробой уровня = настоящий breakout│
  │  vol_z > +2 + нет пробоя    = кульминация       │
  │  vol_z < 0  + тренд         = затухание тренда  │
  │  vol_z < -1 + боковик       = накопление/распр.  │
  └────────────────────────────────────────────────┘

  Breakout подтверждение:
  ──────────────────────────
  1. Цена пробивает наклонную/уровень
  2. vol_z ≥ +1.5 в момент пробоя
  3. OI растёт (новые позиции)
  → Входить по направлению пробоя

  Кульминация:
  ──────────────
  1. vol_z > +2 без нового движения
  2. Длинная тень свечи (отбой)
  3. Funding на экстремуме
  → Ожидать разворот
```

**Важно:** Volume z-score НЕ входит в composite regime (рассчитывается отдельно). Используется как фильтр подтверждения.

## 7. Composite Regime

### Что это
Среднее трёх z-scores: OI + Funding + Liquidations. Показывает общий "температурный режим" рынка по символу.

### Формула

```
composite_z = (oi_zscore + funding_zscore + liq_zscore) / 3
```

> **Ref: TradingRiot** использует 4 компонента (OI + Funding + Skew + VRP) с SMA-5
> сглаживанием. Наш composite использует 3 (OI + Funding + Liq). После реализации
> VRP (см. §13) можно расширить до 4-компонентного composite с опциональным SMA.

### 6 режимов (цветовая шкала)

```
  composite_z
  ─────────────────────────────────────────────────────
     ≤ -2   │ ██ Deep Oversold  │ #22c55e │ зелёный  │
  -2 .. -1  │ ██ Oversold       │ #2dd4bf │ бирюзов. │
  -1 ..  0  │ ██ Neutral Cool   │ #a3e635 │ лайм     │
   0 .. +1  │ ██ Neutral Hot    │ #eab308 │ жёлтый   │
  +1 .. +2  │ ██ Overbought     │ #f97316 │ оранж.   │
     > +2   │ ██ Extreme        │ #ef4444 │ красный  │
  ─────────────────────────────────────────────────────
```

### Как читать

```
  Composite Z
  +3 ┤
     │         ██
  +2 ┤· · · · ██ · · · ·  ← DANGER ZONE
     │       ████
  +1 ┤─ ─ ─ ████ ─ ─ ─ ─
     │     ██████
   0 ┤─────██████─────────
     │   ████████
  -1 ┤─ ████████ ─ ─ ─ ─
     │ ██████████
  -2 ┤██████████ · · · ·  ← OPPORTUNITY ZONE
     │████████████
  -3 ┤
     └────────────────────  время

  Каждый бар окрашен по шкале выше.
  Пунктир на ±2 — границы экстремумов.
```

### Стратегия: Regime Transitions

```
  Переход из красного в оранжевый:
  ──────────────────────────────────
  composite был > +2, стал < +2
  → Рынок начинает остывать
  → Ищем шорт с подтверждением (наклонная, зона)

  Переход из зелёного в бирюзовый:
  ──────────────────────────────────
  composite был < -2, стал > -2
  → Рынок начинает восстанавливаться
  → Ищем лонг на зоне интереса

  Полный цикл:
  ──────────────────────────────────
  Green → Teal → Lime → Yellow → Orange → Red
    ↑ ЛОНГ ЗОНА                    ШОРТ ЗОНА ↑
    │                                         │
    └─────────── цикл повторяется ────────────┘

  Фильтр: НЕ торговать переходы внутри нейтральной зоны (lime↔yellow).
  Только экстремумы (green/red) дают высоковероятные сетапы.
```

## 8. Z-Score Scatter Plots

### Что это
Scatter plot: ось X = z-score метрики в момент T, ось Y = forward return через N дней. Показывает предсказательную силу z-score для будущего движения цены.

### Периоды
Доступны: **10 дней**, **30 дней** (default), **60 дней**.

### Формула forward return

```
return_pct = (price[i + period] - price[i]) / price[i] × 100
```

### Линейная регрессия

```
slope     = (n × Σxy - Σx × Σy) / (n × Σx² - (Σx)²)
intercept = (Σy - slope × Σx) / n
R²        = 1 - SS_res / SS_tot
```

### Как читать

```
  Forward Return (30d)
  +40% ┤
       │   ·        ·
  +20% ┤  · ·  · ·  ·  ·
       │ ·  · ·╱· · · ·  ·    ← линия регрессии
    0% ┤──────╱────────────
       │ · ·╱· · ·  · ·
  -20% ┤  ╱·  · ·     ·
       │╱  ·        ·
  -40% ┤
       └───┴────┴────┴────┴──
       -3  -2   -1    0   +1  +2  +3
                  Z-Score

  ● = историческая точка (серая, opacity 0.15)
  ★ = текущая позиция (жёлтая, #eab308)

  Статистика:
  R² = 0.342  │  n = 245  │  Avg at current: +8.2%
```

### Стратегия: Вероятностный edge

```
  ┌─────────────────────────────────────────────┐
  │  R² > 0.15 → есть статистическая связь      │
  │  R² > 0.30 → сильная связь, можно торговать │
  │  R² < 0.10 → z-score не предсказывает return │
  └─────────────────────────────────────────────┘

  Использование:
  1. Выбрать метрику с наибольшим R²
  2. Посмотреть "Avg at current" — прогноз возврата
  3. Если текущий z < -2 и Avg at current > +10%:
     → Высокая вероятность mean reversion вверх
     → Лонг на зоне интереса
  4. Если текущий z > +2 и Avg at current < -10%:
     → Высокая вероятность коррекции
     → Шорт на наклонной сверху

  Важно: R² считается на исторических данных (365 дней).
  Чем больше n, тем надёжнее статистика.
```

## 9. Liquidation Map

### Что это
Карта теоретических уровней ликвидации на основе текущей цены и распределения плеч. Показывает где сосредоточены стоп-аутные уровни.

### Leverage Tiers

| Плечо | Вес (% от OI) | Liq Long | Liq Short |
|-------|--------------|----------|-----------|
| 5×  | 10% | price × 0.80 | price × 1.20 |
| 10× | 25% | price × 0.90 | price × 1.10 |
| 25× | 30% | price × 0.96 | price × 1.04 |
| 50× | 20% | price × 0.98 | price × 1.02 |
| 100×| 15% | price × 0.99 | price × 1.01 |

### Формулы

```
liq_long_price  = current_price × (1 - 1/leverage)
liq_short_price = current_price × (1 + 1/leverage)
volume_at_level = total_oi_usd × weight
```

### Как читать

```
  Пример: BTC = $100,000, OI = $10B

  Short Liquidations (покупки)    Long Liquidations (продажи)
  ←──────────────────────────────────────────────────────→

  $80k  ███ 5× shorts ($1.0B)
  $90k  ████████ 10× shorts ($2.5B)
  $96k  ██████████ 25× shorts ($3.0B)
  $98k  ███████ 50× shorts ($2.0B)
  $99k  █████ 100× shorts ($1.5B)
        ─────── $100,000 (текущая цена) ───────
  $101k █████ 100× longs ($1.5B)
  $102k ███████ 50× longs ($2.0B)
  $104k ██████████ 25× longs ($3.0B)
  $110k ████████ 10× longs ($2.5B)
  $120k ███ 5× longs ($1.0B)
```

### Стратегия: Ликвидационные магниты

```
  Кластеры ликвидаций работают как магниты для цены.
  Крупный кластер → цена стремится его забрать.

  ┌─────────────────────────────────────────────────┐
  │  1. Определить ближайший крупный кластер         │
  │  2. Кластер сверху = магнит для роста (шорт-сквиз)│
  │  3. Кластер снизу = магнит для падения (лонг-сквиз)│
  │  4. После забора кластера — быстрый разворот     │
  └─────────────────────────────────────────────────┘

  Пример:
  ────────────────────
  BTC $100k, крупный кластер шортов на $104k (25×, $3B)
  → Цена стремится к $104k чтобы ликвидировать шорты
  → На $104k — шорт-сквиз, резкое движение вверх
  → После забора кластера — зона для контр-позиции

  Совмещение с наклонными:
  ────────────────────
  Если ликвидационный кластер совпадает с наклонной:
  → Усиленный магнит
  → Высокая вероятность реакции на этом уровне
  → Идеальная точка для стоп-лосса (за кластером)
```

## 10. Momentum (IV/RV/Skew)

### Что это
Опционные метрики для BTC и ETH (Deribit). IV = ожидаемая волатильность, RV = реализованная, Skew = перекос путов к коллам.

**Только BTC и ETH** — для остальных символов доступен только RV.

### Формулы

**IV (Implied Volatility):** Deribit DVOL Index (30-day)

**RV (Realized Volatility):**
```
log_returns[i] = ln(price[i] / price[i-1])
window = last 30 log returns
variance = Σ(x - mean)² / n          ← population variance
rv_30d = √variance × √365 × 100     ← annualized %
```

**25-Delta Skew:**
```
skew_25d = IV(25δ put) - IV(25δ call)
```
Положительный skew = путы дороже коллов = рынок боится падения.

### Как читать

```
  Volatility (%)
  80 ┤
     │          ╭─╮
  60 ┤    IV ──╯  ╰──    ← IV выше RV: рынок ожидает движение
     │   ╭──────────╮
  40 ┤──╯  RV ──────╰──  ← RV ниже IV: рынок спокойнее ожиданий
     │
  20 ┤
     └──────────────────── время

  Vol Premium = IV - RV
  ──────────────────────
  Premium > 0: опционы дорогие (продавать vol)
  Premium < 0: опционы дешёвые (покупать vol)

  Skew (%)
  +10 ┤
      │    ╭──╮
   +5 ┤───╯  ╰───────    ← путы дороже коллов (страх)
    0 ┤────────────────
   -5 ┤          ╭──╮
      │─────────╯  ╰──   ← коллы дороже путов (жадность)
  -10 ┤
      └──────────────────  время
```

### Стратегия: Vol Premium + Skew Extremes

```
  ┌──────────────────────────────────────────────┐
  │  IV >> RV (premium > 20%)                    │
  │  + skew_z > +2 (путы экстремально дорогие)    │
  │  = Рынок в панике, вероятен разворот вверх    │
  │  → Лонг на зоне интереса                     │
  ├──────────────────────────────────────────────┤
  │  IV << RV (premium < -10%)                   │
  │  + skew_z < -2 (коллы дорогие)               │
  │  = Рынок в эйфории, вероятен разворот вниз   │
  │  → Шорт на наклонной                         │
  ├──────────────────────────────────────────────┤
  │  IV и RV оба низкие (< 30%)                  │
  │  = Сжатие волатильности                      │
  │  → Готовиться к breakout (направление неясно) │
  │  → Ждать подтверждение от OI/Volume           │
  └──────────────────────────────────────────────┘
```

### Крипто-специфика скью (отличие от equities)

В отличие от акций, где skew стабильно положительный (путы дороже из-за crash risk), крипто-skew **часто меняет знак**:
- Положительный skew = спрос на downside protection (страх)
- Отрицательный skew = спекуляция на рост (коллы дороже путов)
- Flip из отрицательного в положительный часто совпадает с локальными топами

> **Ref: TradingRiot** отмечает что крипто-skew "frequently flips" в отличие
> от equities где skew "persistently positive". Это делает skew_z более
> информативным в крипте — экстремумы в обе стороны значимы.

## 11. Orderbook Depth & Skew

### Что это
Глубина ордербука ±2% от mid-price (Binance Futures). Skew показывает дисбаланс между bid и ask сторонами.

### Формулы

```
mid = (best_bid + best_ask) / 2
bid_depth = Σ(price × qty)  для всех bids ≥ mid × 0.98
ask_depth = Σ(price × qty)  для всех asks ≤ mid × 1.02
ob_skew = (bid_depth - ask_depth) / (bid_depth + ask_depth)
```

**Skew range:** [-1, +1]
- +1 = все ордера на bid (максимальная поддержка)
- -1 = все ордера на ask (максимальное давление продаж)
- 0 = баланс

### Как читать

```
  OB Skew
  +1.0 ┤
       │     ╭╮
  +0.5 ┤    ╭╯╰╮         ← больше bidов: поддержка
       │   ╭╯  ╰╮
   0.0 ┤──╯─────╰───────  ← баланс
       │          ╰╮
  -0.5 ┤           ╰─╮    ← больше asков: давление продаж
       │             ╰──
  -1.0 ┤
       └───────────────── время (обновление каждые 30с)

  Z-Score OB Skew (rolling 24h, 288 readings):
  +2 ┤ · · · · · · · ·  ← экстремальная поддержка
  +1 ┤ - - - - - - - -
   0 ┤─────────────────
  -1 ┤ - - - - - - - -
  -2 ┤ · · · · · · · ·  ← экстремальное давление
```

### Стратегия: Подтверждение наклонных

```
  ┌────────────────────────────────────────────────────┐
  │  Цена касается наклонной снизу:                     │
  │  + ob_skew > +0.3 (bid dominance)                  │
  │  + skew_z > +1                                     │
  │  = Ордербук подтверждает отбой от наклонной         │
  │  → Лонг с близким стопом                           │
  ├────────────────────────────────────────────────────┤
  │  Цена касается наклонной сверху:                    │
  │  + ob_skew < -0.3 (ask dominance)                  │
  │  + skew_z < -1                                     │
  │  = Ордербук подтверждает отбой вниз                │
  │  → Шорт с близким стопом                           │
  ├────────────────────────────────────────────────────┤
  │  Пробой наклонной + skew разворот:                  │
  │  Был ob_skew < -0.3, стал > 0 при пробое вверх    │
  │  = Продавцы отступают, покупатели входят            │
  │  → Подтверждение breakout, лонг                    │
  └────────────────────────────────────────────────────┘
```

## 13. Variance Risk Premium (VRP)

> **Статус: TODO** — данные (IV, RV) уже собираются для BTC/ETH. Нужен расчёт VRP и z-score.

### Что это
Разница между implied volatility (ожидание рынка) и realized volatility (фактическое движение). Положительный VRP = опционы переоценены, отрицательный = недооценены.

### Формула

```
VRP = IV_30d - RV_30d
VRP_z = z-score(VRP, window=365)
```

### Почему это работает

VRP стабильно положительный — это "страховая премия". Инвесторы переплачивают за защиту, создавая систематический edge для продавцов vol.

```
  Исторические средние (TradingRiot):
  ──────────────────────────────────────
  SPY:     IV ~18%, RV ~15%, VRP ~3%
  BTC/ETH: IV ~55-70%, RV ~50-65%, VRP ~5-10%

  Крипто VRP выше из-за:
  1. Менее зрелый опционный рынок
  2. Больший спрос на хеджирование
  3. 24/7 торговля (365 дней для annualization)
```

### Как читать

```
  VRP (%)
  +20 ┤
      │       ╭──╮
  +10 ┤  ╭───╯  ╰───   ← VRP высокий: опционы дорогие
      │ ╭╯              = можно продавать vol
   0  ┤╯─────────────── ← нейтраль
      │          ╭─╮
  -10 ┤─────────╯ ╰──  ← VRP отрицательный: опционы дешёвые
      │                  = можно покупать vol
  -20 ┤
      └──────────────── время

  VRP Z-Score:
  z > +2  → VRP экстремально высокий ("rich vol")
  z < -2  → VRP экстремально низкий ("cheap vol")
```

### Стратегия: VRP Mean Reversion

```
  ┌──────────────────────────────────────────────────┐
  │  VRP_z > +2 ("Rich Vol"):                        │
  │  Опционы сильно дороже реального движения        │
  │  → Sell vol: продавать стрэддлы/стрэнглы         │
  │  → Или направленно: VRP_z > +2 + skew_z > +2    │
  │    = панический хедж → лонг на зоне интереса     │
  ├──────────────────────────────────────────────────┤
  │  VRP_z < -2 ("Cheap Vol"):                       │
  │  Опционы дешевле реального движения              │
  │  → Buy vol: покупать стрэддлы перед движением    │
  │  → Или готовиться к breakout (vol expansion)     │
  ├──────────────────────────────────────────────────┤
  │  Важно:                                          │
  │  • Не продавать vol в rising IV — VRP может быть  │
  │    высоким но IV ещё растёт (кластеризация vol)   │
  │  • Ждать признаки mean reversion (IV начал падать)│
  │  • Крипто: VRP > 10% — нормально, не аномалия    │
  └──────────────────────────────────────────────────┘
```

## Appendix A: Формулы

### Z-Score (универсальная)

```
window = последние 365 значений
mean   = Σ(values) / n
std    = √(Σ(x - mean)² / n)     ← population std dev
z      = (current - mean) / std
pctl   = count(x < current) / n × 100

Минимум: n ≥ 7 (иначе z = 0, percentile = 50)
```

### Composite Regime

```
composite_z   = (oi_z + funding_z + liq_z) / 3
composite_pct = (oi_pctl + funding_pctl + liq_pctl) / 3
```

### Percentile Average (Screener)

```
pct_avg = (oi_pctl + funding_pctl + liq_pctl + vol_pctl) / 4
```

### OI Aggregation

```
total_oi = binance_oi_usd
         + bybit_oi_coins × price
         + okx_oi_coins × price
         + bitget_oi_usd
```

### Realized Volatility

```
log_ret[i] = ln(price[i] / price[i-1])
window     = last 30 log returns
rv_30d     = √(Var(window)) × √365 × 100%
```

### 25-Delta Skew

```
skew_25d = IV(25δ put) - IV(25δ call)
```

### Orderbook Skew

```
mid       = (best_bid + best_ask) / 2
bid_depth = Σ(p × q) where p ≥ mid × 0.98
ask_depth = Σ(p × q) where p ≤ mid × 1.02
ob_skew   = (bid - ask) / (bid + ask)
```

### Liquidation Levels

```
liq_long  = price × (1 - 1/leverage)
liq_short = price × (1 + 1/leverage)
```

### Funding Annualized

```
APR = rate_8h × 3 × 365 × 100%
```

### Risk Appetite

```
risk = AVG((oi_z + funding_z + liq_z) / 3) по top-10 по OI
```

### Variance Risk Premium (TODO)

```
VRP    = IV_30d - RV_30d
VRP_z  = z-score(VRP, window=365)
```

### Altcoin OI Dominance (TODO)

```
btc_dom = btc_oi / Σ(all_oi) × 100
alt_dom = 100 - btc_dom
```

### Spot Volume Delta (TODO)

```
delta = taker_buy_vol - taker_sell_vol
```

### Directional Intensity (TODO)

```
DI = (positive_days - negative_days) / total_days   over rolling window
Range: [-1, +1]
```
